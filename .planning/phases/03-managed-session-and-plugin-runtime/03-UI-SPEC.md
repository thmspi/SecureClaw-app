## Component Inventory

Based on CONTEXT.md decisions (D-01 through D-18), this phase requires:

### Page Layout Components
- **Card** ‚Äî Section containers for Session Control, Plugin Runs, Operation History
- **Button** ‚Äî Start/Stop session, Run/Retry plugin actions
- **Separator** ‚Äî Visual division between page sections

- **Badge** ‚Äî Session status indicator (inactive/starting/active/stopping)

### Plugin Runs Components
- **Table** ‚Äî Plugin list with name, status, actions columns
- **Badge** ‚Äî Plugin lifecycle state (Queued, Starting, Running, Completed, Failed)
- **Button** ‚Äî Run action (disabled when no session), Retry action (for failed)
- **Collapsible** ‚Äî Expandable log/progress detail per plugin
- **Progress** ‚Äî Running plugin progress indicator
- **Tooltip** ‚Äî Disabled state reason on hover

### Operation History Components
- **Table** ‚Äî History rows with timestamp, operation type, status, duration
- **Collapsible** ‚Äî Expandable detail panel per operation
- **Select** ‚Äî Filter by operation type
- **Select** ‚Äî Filter by status

## Interaction Contracts

### Page Layout (from D-11, D-12, D-14)
- Single page with three stacked sections: Session Control (top), Plugin Runs (middle), Operation History (bottom)
- Top-right action cluster contains Start/Stop Session only (no other actions)
- Page accessible when prerequisites satisfied (post-install landing)
- Sections use Card components with lg (24px) internal padding

### Session Control (from D-01 through D-06, D-13)
- Session does NOT auto-start on page load
- Start Session CTA prominent (primary variant) when no active session
- Session status badge: `inactive` (muted) ‚Üí `starting` (muted + spinner) ‚Üí `active` (primary/green) ‚Üí `stopping` (muted + spinner)
- Session becomes "active" only after readiness confirmation (not just process spawn)
- Stop button: one-click, shows confirmation dialog, then "stopping" state
- Disabled plugin actions show explicit reason text: "Start a session to run plugins"

### Plugin Runs (from D-07 through D-10)
- Plugin list displayed in table format with columns: Name, Status, Actions
- Run button disabled with tooltip when no active session
- Plugin lifecycle states displayed via Badge:
  - `Queued` ‚Äî muted/gray
  - `Starting` ‚Äî muted + spinner
  - `Running` ‚Äî primary/accent + progress indicator
  - `Completed` ‚Äî success/green checkmark
  - `Failed` ‚Äî destructive/red
- Click row or chevron to expand log/progress detail (Collapsible)
- Retry button appears for Failed plugins (restarts from scratch, preserves history)
- Multiple plugins can run in parallel (capped concurrency, agent discretion on limit)

### Operation History (from D-15 through D-18)
- History persists session start/stop and plugin execution operations
- Summary rows show: timestamp, operation type, status, duration
- Click row to expand details (Collapsible)
- Filter controls inline above table:
  - Operation type: All | Session | Plugin
  - Status: All | Completed | Failed | Running
  - Date: From/To date pickers
- Visual distinction: Session operations use `--muted` badge, Plugin operations use `--secondary` badge
- Retention: last 90 days (backend responsibility)

## State Machines

### Session State Machine

```
[Inactive] --Start--> [Starting] --Ready--> [Active]
                           |                    |
                         Fail               Stop
                           |                    |
                           v                    v
                       [Inactive] <--Done-- [Stopping]
```

States: `inactive`, `starting`, `active`, `stopping`

### Plugin Execution State Machine

```
[Idle] --Run--> [Queued] --Slot Available--> [Starting] --Ready--> [Running]
                                                 |                      |
                                               Fail                 Complete/Fail
                                                 |                      |
                                                 v                      v
                                             [Failed]              [Completed/Failed]
                                                 |
                                              Retry
                                                 |
                                                 v
                                             [Queued]
```

States: `idle`, `queued`, `starting`, `running`, `completed`, `failed`
